# 過去記事 PII 棚卸し (2026-05)

Issue #3 の Phase 1 (read-only 棚卸し) の成果物。

## 棚卸し方針

**重要: 公開済み記事 (zenn.dev で 200 OK 応答) は修正対象外** とする。

理由: Issue 起票時の方針 (in-place push で上書き) から、運用方針を変更したため。
公開済み記事の text 修正は Zenn 上で「更新」扱いとなり、読者・SEO・既読状態に影響を与えうる。
今回の棚卸し結果は **以後の新記事執筆時に同じ漏れを起こさないための知見** として記録する。

スキャン方法・実装は Issue #1 (PR #4) で導入した `hooks/lib/detect-pii.sh` + `~/.config/zenn-pii-blocklist.yaml` をそのまま流用。

```bash
bash hooks/lib/detect-pii.sh articles/*.md 2>&1
```

## 知見・再発防止

### 1. `<username>` の扱いがファイル毎に分かれている

検出された 18 件はすべて以下 2 用途のいずれか:

- **GitHub リポ URL** (`miyashita337/openclaw-rpi5-ops` 等)。これは公開ハンドルなので **本来 PII ではない**
- **CLI プロンプト・ファイルパス例** (`<username>@hostname:~$` 等)。CLAUDE.md 規約では `user@hostname` にマスクすべき

ハンドル名と OS login user 名が同一文字列のため、blocklist の単純一致では context 区別ができない。
→ Issue #6 (false positive 管理ポリシー: 公開ハンドル除外) で対応する。

### 2. `<hostname>` (HOSTNAME) は単独 1 件

記事 #1 (1479b0552927e1.md:92) で `bonjour name` の文脈で実機名が露出。
新記事執筆時は `hostname` または `mymachine.local` にマスクすること。

### 3. `openclaw-07` の集中

12 件と最多。記事 #7 公開時の見落とし。今後の openclaw シリーズ続編では
**push 前に必ず `pre-push hook` を通す** ことで再発を防ぐ (Issue #1 PR #4 で実装済)。
